

# Отговор на въпросите от клиентката

## 1. Повече от един склад — ДА, ИМА ✅
Системата вече поддържа **многоскладов режим**. В секция "Склад" → таб "Складове" може да:
- Активира "Многоскладов режим" (toggle)
- Създаде неограничен брой складове (име, код, адрес, град, телефон)
- Задаване на склад по подразбиране
- Проследяване на наличности по склад (`stock_by_warehouse` таблица)
- Трансфер на стоки между складове (`StockTransferDialog`)

**Извод:** Тази функционалност е готова. Клиентката трябва да активира многоскладовия режим от Склад → Складове и да създаде двата си магазина.

---

## 2. Записване на пратки без касова бележка (ръчни/кеш продажби) — ЧАСТИЧНО ⚠️
Системата има:
- Ръчно добавяне на поръчки (`AddOrderDialog`) — може да се добави поръчка без куриер
- Финансов модул с плащания (платена/неплатена/частично)
- Управление на разходи

**Какво ЛИПСВА:** Няма изричен маркер "Касово плащане / На ръка" в поръчките. Полето `payment_method` в `orders` таблицата не включва опция за кеш/на ръка, и няма флаг "без касова бележка".

**Какво трябва да се добави:**
- Ново поле `payment_method` с опции: "Наложен платеж", "По банка", "На ръка (кеш)", "Карта"
- Филтър по метод на плащане в списъка с поръчки и във финансите
- Отделна статистика за кеш продажби

---

## 3. Печалба за месец/година, статистики — ДА, ИМА ✅
Системата има пълен аналитичен модул:
- **Аналитика** (страница): KPI карти, трендове, ABC анализ, топ продукти, продажби по региони
- **Финанси** (страница): Приходи, разходи, нетна печалба, статуси на плащания
- **RevenueExpenseTrend**: Графика приходи vs разходи по дни и месеци с изчисление на нетна печалба
- Филтриране по период (от-до дата), по магазин, по статус
- Експорт в PDF и Excel

**Извод:** Готово. Клиентката може да отвори Аналитика и да избере период (месец, тримесечие, година).

---

## 4. Автоматично проследяване от Еконт и смяна на статус — ЧАСТИЧНО ⚠️
Системата има:
- ✅ Пълна интеграция с Еконт API (edge function `courier-econt`): създаване на товарителници, проследяване, PDF етикети
- ✅ Ръчно проследяване на пратки (`ShipmentsTab` → бутон "Проследи")
- ✅ `track` action — извиква `getShipmentStatuses` от Еконт API

**Какво ЛИПСВА:**
- ❌ **Автоматично периодично проследяване** (cron/polling) — няма scheduled function, която да проверява статуса на пратките
- ❌ **Автоматична смяна на статус** при доставка — няма логика, която при получен статус "Delivered" от Еконт да промени статуса на поръчката

**Какво трябва да се добави:**
1. Нова edge function `track-shipments-cron` — периодично (на всеки 30 мин) проверява всички активни пратки с Еконт
2. Нова таблица `shipment_tracking_config` — настройка кой статус от Еконт да се маппне към кой статус в системата
3. Настройка в UI за избор на "Еконт статус Доставена → Статус в системата: [избор]"
4. pg_cron или Supabase Cron за автоматично извикване

## План за имплементация (липсващите неща)

### Задача A: Метод на плащане в поръчките
- SQL миграция: добавяне на `payment_method text DEFAULT 'cod'` в `orders`
- Enum стойности: `cod` (наложен платеж), `bank_transfer`, `cash` (на ръка), `card`
- UI: dropdown в AddOrderDialog и EditOrderDialog
- Филтър в OrderFilters

### Задача B: Автоматично Еконт проследяване
- Нова edge function `track-shipments-auto` — взима всички пратки със статус != 'delivered', проверява ги в Еконт API, при промяна обновява shipments + orders
- Нова таблица `courier_tracking_config` — mapping между Еконт статуси и статуси в системата
- UI в Настройки → Куриери → Еконт: "При доставка, промени статуса на: [dropdown]"
- Cron trigger (pg_net или Supabase scheduled function) на всеки 30 минути
